## 📖 第一～三章：基础知识与线性结构

### 1. 以 Niklaus Wirth 的观点，程序等于什么？

**程序 = 数据结构 + 算法**

> Wirth 的经典著作《Algorithms + Data Structures = Programs》提出这一公式，强调：
> - **数据结构**：数据及其组织方式
> - **算法**：操作数据的步骤
> 二者缺一不可，共同构成程序的核心。

---

### 2. 算法的重要特性

| 特性 | 说明 |
|------|------|
| **有穷性 (Finiteness)** | 算法必须在有限步内结束 |
| **确定性 (Definiteness)** | 每一步必须有明确含义，无歧义 |
| **可行性 (Effectiveness)** | 每一步操作都能在有限时间内完成 |
| **输入 (Input)** | 有 0 个或多个输入 |
| **输出 (Output)** | 有 1 个或多个输出 |

---

### 3. 好算法的标准

1. **正确性 (Correctness)** — 算法能正确解决问题
2. **可读性 (Readability)** — 易于理解和维护
3. **健壮性 (Robustness)** — 能处理非法输入
4. **高效性 (Efficiency)** — 时间与空间复杂度低

---

### 4. 线性结构的特点

- **存在唯一第一个元素**（没有前驱）
- **存在唯一最后一个元素**（没有后继）
- **除第一个外**，每个元素有唯一的直接前驱
- **除最后一个外**，每个元素有唯一的直接后继

> 简单说：数据元素之间存在 **一对一(1:1)** 的逻辑关系。

---

### 5. 线性结构与非线性结构的区别

| 对比角度 | 线性结构 | 非线性结构 |
|----------|----------|------------|
| 逻辑关系 | 一对一 (1:1) | 一对多 (1:N) 或多对多 (M:N) |
| 代表 | 线性表、栈、队列 | 树、图 |
| 前驱/后继 | 唯一前驱、唯一后继 | 可能有多个前驱或后继 |

---

### 6. 所学过的线性结构与非线性结构

| 线性结构 | 非线性结构 |
|----------|------------|
| 线性表（顺序表、链表） | 树（二叉树、Huffman树等） |
| 栈 | 图（有向图、无向图） |
| 队列 | 森林 |
| 数组、串 | 集合 |

---

### 7. 头指针、头结点、首元结点的区别

| 概念 | 说明 |
|------|------|
| **头指针 (Head Pointer)** | 指向链表第一个结点的指针，是链表的必要标识 |
| **头结点 (Head Node)** | 在首元结点前附加的一个结点（**不存数据**），**非必需** |
| **首元结点 (First Node)** | 链表中存储第一个数据元素的结点 |

> 🔑 头指针是**必须的**，头结点是**可选的**（带头结点方便统一空表和非空表的操作）。

---

### 8. 带头结点和不带头结点的线性链表区别

| 对比项 | 带头结点 | 不带头结点 |
|--------|----------|------------|
| 空表判断 | `L->next == NULL` | `L == NULL` |
| 插入/删除操作 | 不需要单独处理首元结点 | 插入/删除首元结点需特殊处理 |
| 代码统一性 | 操作统一，更简洁 | 操作需分情况讨论 |

---

### 9. 单链表、双链表、循环链表的区别及优缺点

| 类型 | 指针域 | 优点 | 缺点 |
|------|--------|------|------|
| **单链表** | 1个 next 指针 | 存储密度高，节省空间 | 只能单向遍历，找前驱需从头 |
| **双链表** | 2个指针 (prior, next) | 可双向遍历，找前驱方便 | 占用更多空间，操作稍复杂 |
| **循环链表** | 尾结点指向头结点 | 可从任一结点遍历全表 | 防死循环需注意终止条件 |

> 💡 **循环双链表** = 双链表 + 循环，兼具两者优点。

---

### 10. 栈和队列是什么样的线性表？

- **栈 (Stack)**：**后进先出 (LIFO)** 的线性表。只能在表尾（栈顶）进行插入和删除。
- **队列 (Queue)**：**先进先出 (FIFO)** 的线性表。在一端（队尾）插入，另一端（队头）删除。

---

### 11. 顺序线性表、顺序栈、顺序队列的区别

| 结构 | 操作限制 | 插入/删除位置 |
|------|----------|--------------|
| 顺序表 | 无限制 | 任意位置 |
| 顺序栈 | LIFO | 仅栈顶（表尾） |
| 顺序队列 | FIFO | 队尾插入，队头删除 |

---

### 12. 栈和队列的应用实例

**栈的应用：**
- 函数调用/递归调用（系统栈）
- 表达式求值（操作数栈、运算符栈）
- 括号匹配检查
- 浏览器的"后退"功能
- 撤销操作 (Ctrl+Z)

**队列的应用：**
- 打印机任务队列
- CPU 进程调度（先来先服务FCFS）
- 消息队列（如 RabbitMQ、Kafka）
- 树的层次遍历
- BFS（广度优先搜索）

---

### 13. 解决队列和栈"溢出"的方法

#### 栈的溢出解决

| 问题 | 方法 |
|------|------|
| **上溢 (Overflow)** | ① 重新分配更大空间（动态扩容），将原数据复制到新空间 ② 使用链式存储结构（无需预先分配空间） |
| **下溢 (Underflow)** | 正常情况，表示栈已空，可作为递归/算法终止条件 |

#### 队列的溢出解决

| 问题 | 方法 |
|------|------|
| **上溢 (Overflow)** | ① 重新分配更大空间（动态扩容） ② 使用链式存储结构（链队列无容量限制） ③ **循环队列**：用 `(rear+1) % MAXSIZE == front` 判断队满，留一个空位区分空/满，重用已出队的空间 |
| **下溢 (Underflow)** | 正常情况，表示队列已空，可作为遍历/调度终止条件 |

> **总结**：栈和队列的上溢通用解法是扩容或改用链式存储。队列独有的优化是**循环队列**，通过取模运算将顺序结构逻辑上变为环形，避免"假溢出"（有空位但因 front 在前端而无法入队）。

---

### 14. 循环队列的循环是如何实现的？

通过 **取模运算（%）** 实现：

```c
// 入队
rear = (rear + 1) % MAXSIZE;
// 出队
front = (front + 1) % MAXSIZE;
// 判断队满
(rear + 1) % MAXSIZE == front;
// 队列长度
(rear - front + MAXSIZE) % MAXSIZE;
```

> 将顺序队列在逻辑上视为一个**环形空间**，当 rear/front 到达数组末尾时自动"折回"到开头，从而重用已出队的空间。

### 15. 对称矩阵、三角矩阵的压缩存储

**对称矩阵 (n×n)：**
- 只需存储上三角或下三角（含对角线），共 `n(n+1)/2` 个元素
- 按下标映射：`k = i(i-1)/2 + j-1` (下三角，行优先)

```c
// ==================== 对称矩阵 ====================
// 输入对称矩阵（只输入下三角，行优先压缩存储）
void inputSymmetric(int *arr, int n) {
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= i; j++)              // 只遍历下三角 + 对角线
            scanf("%d", &arr[i*(i-1)/2 + j-1]);
}

// 获取对称矩阵元素值（利用 a[i][j] == a[j][i]）
int getSymmetric(int *arr, int n, int i, int j) {
    if (i >= j)
        return arr[i*(i-1)/2 + j-1];   // 下三角部分，直接映射
    else
        return arr[j*(j-1)/2 + i-1];   // 上三角部分，交换 i,j 后映射
}
```

**下三角矩阵：**
- 存储下三角（含对角线），共 `n(n+1)/2 + 1` 个元素（多一个存常数）
- 下标映射（行优先，0-based 数组下标 k，1-based 行列 i,j）：
  - **下三角部分 (i ≥ j)**：`k = i(i-1)/2 + j-1`
  - **上三角部分 (i < j)**：全都是同一个常数 c（单独存）

**上三角矩阵：**
- 存储上三角（含对角线），共 `n(n+1)/2 + 1` 个元素（多一个存常数）
- 下标映射（行优先）：
  - **上三角部分 (i ≤ j)**：`k = (i-1)(2n-i+2)/2 + (j-i)`
  - **下三角部分 (i > j)**：全都是同一个常数 c（单独存）

```c
// ==================== 下三角矩阵 ====================
// 输入下三角矩阵（行优先压缩存储）
void inputLowerTriangular(int *arr, int n) {
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= i; j++)              // 只遍历下三角 + 对角线
            scanf("%d", &arr[i*(i-1)/2 + j-1]);
    scanf("%d", &arr[n*(n+1)/2]);                  // 最后一个位置存常数 c
}

// 获取下三角矩阵元素值
int getLowerTriangular(int *arr, int n, int i, int j) {
    if (i >= j)
        return arr[i*(i-1)/2 + j-1];   // 下三角部分，直接映射
    else
        return arr[n*(n+1)/2];          // 上三角常数 c（存在数组末尾）
}

// ==================== 上三角矩阵 ====================
// 输入上三角矩阵（行优先压缩存储）
void inputUpperTriangular(int *arr, int n) {
    for (int i = 1; i <= n; i++)
        for (int j = i; j <= n; j++)              // 只遍历上三角 + 对角线
            scanf("%d", &arr[(i-1)*(2*n-i+2)/2 + (j-i)]);
    scanf("%d", &arr[n*(n+1)/2]);                  // 最后一个位置存常数 c
}

// 获取上三角矩阵元素值
int getUpperTriangular(int *arr, int n, int i, int j) {
    if (i <= j)
        return arr[(i-1)*(2*n-i+2)/2 + (j-i)];  // 上三角部分，直接映射
    else
        return arr[n*(n+1)/2];                    // 下三角常数 c
}
```

### 19. 二叉树的性质

- **第 i 层**最多结点数：**2^(i-1)** (i ≥ 1)
- **深度为 K**的二叉树最多结点数：**2^K - 1** (K ≥ 1)
- **满二叉树**：所有层结点数都达到最大值
- **完全二叉树**：除最后一层外全满，且最后一层结点靠左排列

---

### 20. 叶子结点数 n₀ 与度为2的结点数 n₂ 的关系

**n₀ = n₂ + 1**

> 证明：设总结点数 n = n₀ + n₁ + n₂
> 总边数 = n - 1 = n₁ + 2n₂
> 联立得：n₀ + n₁ + n₂ - 1 = n₁ + 2n₂ → n₀ = n₂ + 1

🔑 这是最重要的一棵树的性质之一，常用于递归计算。

### 21. n 个结点的完全二叉树的深度

**深度 = ⌊log₂n⌋ + 1**（向下取整 + 1）

### 22. 在二叉树的顺序存储结构中如何求结点的双亲、孩子？

用一维数组存储，利用下标关系确定关系：

| 关系 | 公式（根从 1 开始） |
|------|-------------------|
| 结点 i 的**左孩子** | `2i` |
| 结点 i 的**右孩子** | `2i + 1` |
| 结点 i 的**双亲** | `⌊i/2⌋` |
| 判断是否为**叶子** | `2i > n` |

> 适用于完全/满二叉树，普通二叉树浪费空间（需用特殊值补空缺）。

### 23. 二叉链表与三叉链表的空链域数

**n 个结点的二叉树：**

| 存储结构 | 总指针数 | 非空指针 (边数) | 空链域数 |
|----------|---------|-----------------|---------|
| **二叉链表** | 2n | n - 1 | **n + 1** |
| **三叉链表** | 3n | n - 1（parent也非空时另有 n-1） | **n + 2** |

> 🔥 常见考题：n 个结点的二叉链表有 **n + 1** 个空链域，这些空链域被线索化利用。

### 24. 线索化二叉树

**为什么可以不增加指针域？**
> 利用 n+1 个空链域来存放前驱/后继指针（线索），无需额外空间。

**线索化的目的：**
1. 加快遍历速度（无需递归/栈）
2. 快速找到结点的前驱和后继
3. 简化遍历算法

---

### 25. 如何识别线索化二叉树的指针域？

在结点中增加两个标志位：

```c
typedef struct ThreadNode {
    ElemType data;
    struct ThreadNode *lchild, *rchild;
    int ltag, rtag;  // 0: 指向孩子  1: 指向线索（前驱/后继）
} ThreadNode;
```

- `ltag == 0` → lchild 指向**左孩子**
- `ltag == 1` → lchild 指向**前驱结点**
- `rtag == 0` → rchild 指向**右孩子**
- `rtag == 1` → rchild 指向**后继结点**

### 26. 树的三种存储结构优缺点

| 存储结构 | 优点 | 缺点 | 适用场景 |
|----------|------|------|----------|
| **双亲表示法** | 找双亲快 O(1) | 找孩子慢，需遍历 | 仅需查双亲的场景 |
| **孩子表示法** | 找孩子快 | 找双亲慢 | 仅需查孩子的场景 |
| **孩子兄弟表示法** | 灵活，可转二叉树 | 查找略复杂 | **最常用**，森林←→二叉树转换 |

### 27. 森林←→二叉树的转换

**孩子兄弟表示法**可将森林转为二叉树。

**转换规则：**
- 每棵树的根结点**互视为兄弟**
- 树结点：第一个孩子 → 左子树，下一个兄弟 → 右子树

**各域注释：**
- `firstChild` — 指向第一个孩子
- `nextSibling` — 指向下一个兄弟
- `data` — 结点数据

**找到森林的 n 棵树：**
> 从根结点开始，沿 `nextSibling` 链不断右移直到 NULL，个数即为树的数量。

### 29. 写算法求树中结点的度；树的度；树中的叶子结点数；树中的非终端结点数；树中某结点的兄弟、祖先、子孙、层次、堂兄弟；树的高度；森林中树的数目

> 以下算法均基于**孩子兄弟表示法**（孩子-兄弟链表），将一般树用二叉树形式存储。
> 每个结点有两个指针：`firstChild`（指向第一个孩子）和 `nextSibling`（指向下一个兄弟）。

**存储结构定义：**

```c
typedef int ElemType;
typedef struct TreeNode {
    ElemType data;                  // 结点数据
    struct TreeNode *firstChild;    // 指向第一个孩子
    struct TreeNode *nextSibling;   // 指向下一个兄弟
} TreeNode, *Tree;
```

---

#### ① 求树中结点的度（某结点的孩子个数）

```c
// 算法：遍历结点的 firstChild 链，沿 nextSibling 统计孩子个数
int nodeDegree(TreeNode *node) {
    if (node == NULL) return 0;
    int cnt = 0;
    TreeNode *p = node->firstChild;
    while (p != NULL) {
        cnt++;
        p = p->nextSibling;
    }
    return cnt;
}
// 使用示例：
// int d = nodeDegree(t);  // 返回结点 t 的度
```

---

#### ② 求树的度（树中所有结点度的最大值）

```c
// 算法：递归遍历树中每个结点，取 max(当前结点度, 各子树度的最大值)
int treeDegree(Tree T) {
    if (T == NULL) return 0;
    int maxDeg = nodeDegree(T);           // 当前结点的度
    TreeNode *p = T->firstChild;
    while (p != NULL) {
        int childDeg = treeDegree(p);     // 递归求各子树的度
        if (childDeg > maxDeg) maxDeg = childDeg;
        p = p->nextSibling;
    }
    return maxDeg;
}
// 使用示例：
// int maxD = treeDegree(T);  // 返回整棵树的度
```

---

#### ③ 求树中的叶子结点数（度=0的结点数）

```c
// 算法：递归遍历，若 firstChild==NULL 则是叶子，计数+1
int countLeaves(Tree T) {
    if (T == NULL) return 0;
    // 如果没有第一个孩子，说明度为0，是叶子
    if (T->firstChild == NULL) return 1;
    // 否则递归遍历所有孩子（沿兄弟链走）
    int cnt = 0;
    TreeNode *p = T->firstChild;
    while (p != NULL) {
        cnt += countLeaves(p);
        p = p->nextSibling;
    }
    return cnt;
}
// 使用示例：
// int leaves = countLeaves(T);  // 返回叶子结点数
```

---

#### ④ 求树中的非终端结点数（度≠0的结点数）

```c
// 算法：递归遍历，若 firstChild!=NULL 则是非终端结点，计数+1
int countNonTerminal(Tree T) {
    if (T == NULL) return 0;
    // 如果没有孩子，是终端结点，不计入
    if (T->firstChild == NULL) return 0;
    // 当前结点是非终端结点(1) + 所有子树中的非终端结点
    int cnt = 1;
    TreeNode *p = T->firstChild;
    while (p != NULL) {
        cnt += countNonTerminal(p);
        p = p->nextSibling;
    }
    return cnt;
}
// 使用示例：
// int nonTerm = countNonTerminal(T);
// 验证：countLeaves(T) + countNonTerminal(T) == countNode(T)
```

#### ⑤ 求树中某结点的兄弟

```c
// 算法：兄弟即 parent->firstChild 链上除自身外的其他结点
// 思路：先找到指定结点的父结点，再从父结点的 firstChild 开始输出所有兄弟

// 查找某结点在树中的父结点（孩子兄弟表示法中需遍历）
TreeNode *findParent(Tree T, TreeNode *target) {
    if (T == NULL || target == NULL) return NULL;
    // 遍历所有孩子，检查是否是目标结点的父结点
    TreeNode *p = T->firstChild;
    while (p != NULL) {
        if (p == target) return T;           // 找到了，T 就是父结点
        TreeNode *res = findParent(p, target); // 递归在子树中找
        if (res != NULL) return res;
        p = p->nextSibling;
    }
    return NULL;
}

// 输出某结点的所有兄弟（不包括自身）
void printSiblings(Tree T, TreeNode *target) {
    if (T == NULL || target == NULL) return;
    // 根结点无兄弟
    if (target == T) {
        printf("根结点没有兄弟\n");
        return;
    }
    TreeNode *parent = findParent(T, target);
    if (parent == NULL) {
        printf("结点不在树中\n");
        return;
    }
    printf("兄弟结点: ");
    TreeNode *p = parent->firstChild;
    while (p != NULL) {
        if (p != target) printf("%d ", p->data);
        p = p->nextSibling;
    }
    printf("\n");
}

// printSiblings(T, someNode);
```

#### ⑥ 求树中某结点的祖先（从根到父结点路径上的所有结点）

```c
// 算法：从根出发递归查找目标结点，用数组记录路径上的结点
// 找到目标后，路径上除目标自身外的所有结点都是祖先

int findAncestors(Tree T, TreeNode *target, TreeNode *path[], int depth) {
    if (T == NULL || target == NULL) return 0;

    path[depth] = T;  // 当前结点加入路径

    // 找到了目标结点
    if (T == target) {
        printf("祖先结点（从根到父）: ");
        for (int i = 0; i < depth; i++) {  // depth-1 是父结点位置

        return 1;  // 找到了

    TreeNode *p = T->firstChild;
    while (p != NULL) {
        if (findAncestors(p, target, path, depth + 1)) return 1;
        p = p->nextSibling;
    }
    return 0;  // 没找到
}
```

#### ⑨ 求树中某结点的堂兄弟（同一层但父结点不同的结点）

```c
// 算法：堂兄弟 = 同层中父结点不是目标结点父结点的那些结点
// 思路：先求目标结点的层次和父结点，再层序遍历整棵树，
//       收集同层且父结点不同的结点

void printCousins(Tree T, TreeNode *target) {
    if (T == NULL || target == NULL || T == target) {
        printf("根结点没有堂兄弟\n");
        return;
    }

    // 1. 找到目标结点的父结点和层次
    TreeNode *targetParent = findParent(T, target);
    int targetLevel = getNodeLevel(T, target, 1);

    // 2. 层序遍历树，收集同一层中父结点不是 targetParent 的结点
    //    用队列进行层序遍历，同时记录每个结点的父结点和层次
    typedef struct {
        TreeNode *node;
        TreeNode *parent;
        int level;
    } QueueItem;

    QueueItem queue[200];
    int front = 0, rear = 0;

    // 根结点入队
    queue[rear++] = (QueueItem){T, NULL, 1};

    printf("堂兄弟结点: ");
    int found = 0;

    while (front < rear) {
        QueueItem cur = queue[front++];

        // 如果是同一层且父结点不同，就是堂兄弟
        if (cur.level == targetLevel && cur.parent != targetParent && cur.node != target) {
            printf("%d ", cur.node->data);
            found = 1;
        }

        // 如果层次已经超过目标层次，可以提前结束
        if (cur.level > targetLevel) break;

        // 孩子入队
        TreeNode *child = cur.node->firstChild;
        while (child != NULL) {
            queue[rear++] = (QueueItem){child, cur.node, cur.level + 1};
            child = child->nextSibling;
        }
    }

    if (!found) printf("（无堂兄弟）");
    printf("\n");
}
// 使用示例：
// printCousins(T, someNode);
```

```c
        p = p->nextSibling;  // 下一棵树的根
    }
    return cnt;
}
```

### 30. Huffman 树

**解决的问题：** 数据压缩（最小带权路径长度编码）

**构造方法：**
1. 将所有结点按权值放入最小堆/有序列表
2. 取两个权值最小的结点作为左右子树，合并为新结点（权值和）
3. 将新结点放回
4. 重复 2-3，直到只剩一个结点（即 Huffman 树的根）

> **Huffman 编码**：左分支为 0，右分支为 1，从根到叶子的路径就是该字符的编码。
> 特点是**前缀编码**（任何编码都不是另一编码的前缀），保证无歧义解码。

### 32. 邻接矩阵（图的数组表示法）

**定义：** 用一个 `n×n` 的二维数组 `adj[n][n]` 存储顶点间的邻接关系。

- 若顶点 i 到顶点 j **有边**，则 `adj[i][j] = 1`
- 若顶点 i 到顶点 j **无边**，则 `adj[i][j] = 0`

**三种图的矩阵特点：**
- **无向图**：边没有方向，(i,j) 和 (j,i) 是同一条边，所以矩阵是**对称矩阵**（`adj[i][j] == adj[j][i]`），对角线为 0（无自环时）
- **有向图**：边有方向，不一定对称——第 i **行**表示从 i **出去**的边，第 j **列**表示**进入** j 的边
- **带权图（网）**：`adj[i][j]` 存权值 w，无边存 ∞，对角线存 0

**性质：**
| | 无向图 | 有向图 |
|---|---|---|
| 第 i 行非零个数 | 顶点 i 的度 | 顶点 i 的出度 |
| 第 i 列非零个数 | 顶点 i 的度 | 顶点 i 的入度 |
| 非零元总数 | 2×边数 | 边数 |
| 空间复杂度 | O(n²) | O(n²) |

> **局限：** 邻接矩阵要求顶点编号必须是 `0 ~ n-1` 的连续整数。
> 
> - **编号不连续**（如 1, 2, 4, 5, 8, 9）→ 不能直接用，否则数组下标 0、3、6、7 全空着
> - **顶点是字符串**（如"北京"、"上海"）→ 更不能用，数组下标必须是整数
> 
> **解决：** 额外建一张映射表，把任意顶点标识转成连续下标：
> 
> ```
> 原始编号:  1   2   4   5   8   9
> 映射下标:  0   1   2   3   4   5
> ```
> 
> 查找、遍历时先通过映射表把顶点转成下标，再去访问 `adj[i][j]`。

### 33. 求图中结点的度

**无向图：**
- 结点的度 = 关联该结点的边数（邻接矩阵中行/列之和）

**有向图：**
- **入度** = 指向该结点的边数（列之和）
- **出度** = 从该结点出发的边数（行之和）
- **度** = 入度 + 出度

// 参数：adj-邻接矩阵  n-顶点个数  v-要查的顶点（从0开始编号）
```c
// ==================== 无向图 ====================
// 邻接矩阵求结点度（行之和 = 列之和）
int degree(int adj[][MAX], int n, int v) {
    int count = 0;
    for (int i = 0; i < n; i++)
        if (adj[v][i]) count++;
    return count;
}

// ==================== 有向图 ====================
// 出度：第 v 行之和（从 v 出发的边）
int outDegree(int adj[][MAX], int n, int v) {
    int count = 0;
    for (int j = 0; j < n; j++)
        if (adj[v][j]) count++;
    return count;
}

// 入度：第 v 列之和（指向 v 的边）
int inDegree(int adj[][MAX], int n, int v) {
    int count = 0;
    for (int i = 0; i < n; i++)
        if (adj[i][v]) count++;
    return count;
}

// 有向图结点的（总）度 = 入度 + 出度
int degreeDirected(int adj[][MAX], int n, int v) {
    return outDegree(adj, n, v) + inDegree(adj, n, v);
}
```

### 37. Prim 与 Kruskal 算法过程（图示说明）

**Prim 算法过程：**
```
① 任选起始顶点 A
② 找与当前集合相连的最小权边
③ 将顶点加入集合
④ 重复 ②③ 直到所有顶点加入
```

**Kruskal 算法过程：**
```
① 将所有边按权值从小到大排序
② 选最小边，若不形成环则加入
③ 重复 ② 直到选出 n-1 条边
```

### 40. Dijkstra vs Floyd 算法

| 算法 | 用途 | 时间复杂度 | 优点 | 缺点 |
|------|------|-----------|------|------|
| **Dijkstra** | 单源最短路径 | O(n²) | 效率高，适合单源 | 不能处理负权边 |
| **Floyd** | 所有顶点间最短路径 | O(n³) | 可处理负权边，代码简洁 | 效率低，空间大 |

**Dijkstra 思想：** 按路径长度递增顺序产生最短路径（贪心）
**Floyd 思想：** 动态规划，逐步允许经过更多中间结点，迭代更新最短距离

### 42. 折半查找判定树与 ASL

### 46. 哈希表详解

**哈希函数构造方法：**
1. **直接定址法** — f(key) = a·key + b
2. **除留余数法** — f(key) = key % p（p ≤ m 的质数）
3. **数字分析法** — 取关键字的若干位
4. **平方取中法** — key² 的中间几位
5. **折叠法** — 将 key 分割后叠加

**选取哈希函数的因素：**
- 关键字分布
- 哈希表长度
- 计算效率
- 冲突处理方法

**开放定址法：** `Hi = (H(key) + di) % m`
- 线性探测：di = 1, 2, 3, ...
- 平方探测：di = 1², -1², 2², -2², ...
- 再哈希法：di = Hash₂(key)

**哈希表适用场合：**
> 关键字范围大但实际记录少，需要**快速查找**（O(1) 期望时间）的场景。

**决定 ASL 的因素：**
- 哈希函数质量
- 冲突处理方法
- **装填因子 α = n/m**（最关键）

> 🔑 ASL 只与装填因子 α 有关，与 n 无关！

### 47. 排序算法的稳定性

| 稳定的排序 | 不稳定的排序（及反例） |
|-----------|---------------------|
| ✅ **冒泡排序** | ❌ **快速排序** — 相同值可能交换 |
| ✅ **直接插入排序** | ❌ **希尔排序** — 分组排序破坏稳定性 |
| ✅ **归并排序** | ❌ **堆排序** — 父子交换破坏稳定性 |
| ✅ **基数排序** | ❌ **简单选择排序** — 远距离交换 |

> **反例说明不稳定：** 序列 [5a, 3, 5b, 1]
> - 快排后可能得到 [1, 3, 5b, 5a]，5a 和 5b 相对顺序改变 → 不稳定

### 48. 堆排序

**什么是堆？**
- **大根堆**：每个结点 ≥ 左右孩子（根最大）
- **小根堆**：每个结点 ≤ 左右孩子（根最小）

**堆排序存储结构：** 一维数组（顺序存储完全二叉树）

**递增/递减：** 
- **大根堆** → 递增排序（每次把最大值放末尾）
- **小根堆** → 递减排序

**建堆时间复杂度：** O(n)

**建堆过程：**
```
① 将数组视为完全二叉树
② 从最后一个非叶子结点开始
③ 向下调整（筛选），使子树满足堆性质
④ 依次向前处理每个非叶子结点
⑤ 全部调整完 → 建成堆
```

### 49. 基于比较的排序最好时间复杂度

> **O(n·log n)**

基于比较的排序算法在最坏情况下能达到的最好时间复杂度就是 **O(n·log n)**，这是信息论下界（决策树模型证明）。

达到此下界的算法：**归并排序、堆排序、快速排序（平均）**。

### 50. 各种排序算法的适用场合

| 算法 | 适合场合 |
|------|---------|
| **直接插入排序** | 基本有序、小规模数据 |
| **冒泡排序** | 基本有序、代码简单 |
| **快速排序** | 一般情况、大规模随机数据（平均最快） |
| **堆排序** | 大规模数据、需要最坏情况保证、取 Top-K |
| **基数排序** | 位数固定的整数/字符串排序、时间复杂度受关键字范围影响 |

### 51. 各种排序算法复杂度汇总

| 排序算法 | 平均时间 | 最坏时间 | 最好时间 | 空间 | 稳定 |
|----------|---------|---------|---------|------|------|
| 直接插入 | O(n²) | O(n²) | O(n) | O(1) | ✅ |
| 希尔排序 | O(n^1.3) | O(n²) | O(n) | O(1) | ❌ |
| 冒泡排序 | O(n²) | O(n²) | O(n) | O(1) | ✅ |
| 快速排序 | **O(n·log n)** | O(n²) | O(n·log n) | O(log n) | ❌ |
| 简单选择 | O(n²) | O(n²) | O(n²) | O(1) | ❌ |
| **堆排序** | **O(n·log n)** | **O(n·log n)** | **O(n·log n)** | O(1) | ❌ |
| **归并排序** | **O(n·log n)** | **O(n·log n)** | **O(n·log n)** | O(n) | ✅ |
| 基数排序 | O(d(n+r)) | O(d(n+r)) | O(d(n+r)) | O(r) | ✅ |

> **d** — 关键字位数，**r** — 基数，**n** — 记录数

### 52. 《数据结构》课程全面总结

**核心思想：** 合理组织数据，设计高效算法

**三大主线：**

```
逻辑结构 → 存储结构 → 基本运算 → 算法性能分析
```

**四大逻辑结构：**
1. **集合** — 元素间无关系（最弱）
2. **线性结构** — 一对一（线性表、栈、队列、串）
3. **树形结构** — 一对多（树、二叉树、Huffman树）
4. **图形结构** — 多对多（有向图、无向图、网）

**两大操作类型：**
- **查找**：顺序查找、折半查找、BST、AVL、哈希
- **排序**：插入、交换、选择、归并、基数

**衡量标准：**
- **时间复杂度** — 运行时间（最重要）
- **空间复杂度** — 内存消耗
- **稳定性** — 排序算法中相同元素的相对顺序

### 二、必背算法 (手写)

```c
// 1. 二叉树中序遍历（递归）
void inorder(BiTree T) {
    if (T) {
        inorder(T->lchild);
        visit(T);
        inorder(T->rchild);
    }
}

// 2. 二叉树层次遍历（队列）
void levelOrder(BiTree T) {
    Queue Q; initQueue(&Q);
    enQueue(&Q, T);
    while (!isEmpty(Q)) {
        BiTree p = deQueue(&Q);
        visit(p);
        if (p->lchild) enQueue(&Q, p->lchild);
        if (p->rchild) enQueue(&Q, p->rchild);
    }
}

// 3. 快速排序
void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pivot = partition(arr, low, high);
        quickSort(arr, low, pivot - 1);
        quickSort(arr, pivot + 1, high);
    }
}

// 4. 折半查找
int binarySearch(int arr[], int n, int key) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = (low + high) / 2;
        if (arr[mid] == key) return mid;
        else if (arr[mid] < key) low = mid + 1;
        else high = mid - 1;
    }
    return -1;
}
```

### 三、易混淆概念辨析

| 易混组合 | 关键区别 |
|----------|---------|
| 栈 vs 堆 | 栈是线性结构 LIFO；堆是完全二叉树（数据结构）或内存管理区域 |
| 完全二叉树 vs 满二叉树 | 满二叉树全满；完全二叉树只最后一层不满且靠左 |
| 线索二叉树 vs 二叉链表 | 线索二叉树用空链域加标志位存前驱后继 |
| 最小生成树 vs 最短路径 | 前者连通所有顶点的最小代价树；后者是某两点的最短路径 |
| 深度优先 vs 广度优先 | DFS 用栈（递归）；BFS 用队列 |
| 稳定排序 vs 不稳定排序 | 相同值排序后相对顺序是否改变 |
